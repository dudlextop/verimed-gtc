# Analytics V2 — реализация

## Объём фазы

Фаза `design-v2-analytics` переносит на V2 только:

- рабочую главную `/`;
- полноэкранный управленческий обзор `/overview`;
- региональный мониторинг на статической геометрии Казахстана;
- динамику сигналов из существующего overview API;
- печать, полноэкранное действие и состояния двух аналитических экранов.

Backend, формулы, база, миграции, существующие рабочие маршруты и остальные продуктовые страницы не изменялись.

## Главная

Главная остаётся рабочим стартом, а не полным аналитическим экраном. Тёмный CommandCenter заменён светлой V2-композицией:

- четыре компактных показателя текущего рабочего контура;
- одна рекомендация на основе приоритетной организации;
- одна primary CTA `Перейти к проверке`;
- вторичный переход `Открыть аналитический обзор`;
- сохранён прямой переход к приоритетной организации.

Ниже показываются ровно три объекта: сигнал, организация и повторяющаяся модель. Их показатели используют разные `DomainIndicator`: приоритет для сигнала и организации, важность и устойчивость для модели. Блок изменений стал компактным, а качество анализа и экспертный контур перенесены на второй уровень disclosure.

Главная больше не дублирует карту и динамику overview. Загрузка использует dashboard skeleton, ошибка — общее русскоязычное состояние с повторной попыткой, отсутствие приоритетных объектов — три честных пустых состояния.

## Аналитический обзор

`/overview` сохраняет отдельную полноэкранную оболочку без Sidebar, но теперь использует общий `BrandLogo`, Inter, V2-токены, семантические показатели и графическую тему.

Последовательность:

1. контекст анализа и действия;
2. главный вывод из API и управленческие показатели;
3. три приоритетных объекта;
4. карта и динамика — две ведущие визуализации;
5. структурированные выводы;
6. качество анализа;
7. компактное распределение повторяющихся моделей;
8. изменения;
9. одно рекомендуемое действие.

`schema_version < 2` не ломает страницу: обзор показывает предупреждение об ограниченном региональном контракте. Отсутствие timeline или ошибка геометрии обрабатываются локально, поэтому остальные данные остаются доступны.

## Карта Казахстана

Источник геометрии — локальный asset `frontend/public/maps/kazakhstan-adm1.geojson`. Компонент не обращается к внешнему map provider и не использует API key.

`frontend/lib/kazakhstan-geometry.ts`:

- проверяет структуру GeoJSON;
- преобразует Polygon и MultiPolygon в стабильные SVG paths;
- использует одну проекцию для всей коллекции;
- не изменяет feature IDs.

`RegionalMonitoringMap` сопоставляет API и геометрию только через typed mapping из `frontend/lib/region-map.ts`. Fuzzy matching отсутствует. Шымкент (`KZ-SHY`) намеренно не получает геометрию другой области. Неизвестный код остаётся в ранжированном списке и не окрашивает случайный регион.

Регион без агрегата:

- имеет нейтральную заливку;
- обозначен как `Нет данных`;
- не участвует в шкале интенсивности;
- не интерпретируется как нулевое значение.

Фокус или hover региона открывает компактный фактический контекст: сигналы, уникальные записи, финансовую значимость, организации, максимальный приоритет и ведущую организацию. Активация ведёт в `/organizations` с существующим `region` filter.

## Переключатель слоя

Одна карта поддерживает три слоя:

- `Сигналы`;
- `Финансовая значимость`;
- `Приоритет`.

Переключатель реализован как keyboard-accessible `radiogroup`: работают click, Arrow Left/Right, Home и End. Состояние обозначается фоном, текстом и `aria-checked`, а не только цветом. На узком экране длинная визуальная подпись сокращается до `Финансы`, сохраняя полное доступное имя.

## Mobile region list

Ранжированный список регионов существует независимо от canvas и является главным мобильным представлением. Он показывает:

- место;
- полное название региона;
- число организаций;
- выбранный показатель;
- переход к отфильтрованным организациям.

На 375 px карта раскрывается вторым уровнем через `Показать карту`. Ошибка геометрии не скрывает ранжированный список.

## Динамика сигналов

`OverviewSignalTimeline` получает только `timeline` из `/api/analytics/overview`.

- порядок API сохраняется;
- отсутствующие периоды не достраиваются;
- используется линейная линия без сглаживания и анимации;
- единственный ряд — фактическое число сигналов;
- предусмотрены error, empty и insufficient states;
- текстовая альтернатива перечисляет каждый период и значение.

После visual QA текстовая альтернатива была заменена с visually-hidden table на компактный скрытый текст: семантика сохранилась, а таблица больше не создаёт горизонтальное переполнение на 375 px.

## Print и fullscreen

Печать использует стандартный browser print:

- светлый фон;
- BrandLogo, период и время расчёта;
- скрытые интерактивные действия;
- защищённые от разрыва ключевые секции;
- видимая карта и текстовые альтернативы;
- сохранённая подпись о синтетическом наборе;
- ссылки не разворачиваются в длинные URL.

Fullscreen использует существующий Browser Fullscreen API. Действие инициируется пользователем, отслеживает `fullscreenchange`, сохраняет стандартный выход по Escape и возвращает фокус trigger-элементу. При недоступном API показывается короткий fallback без отдельного режима продукта.

## Удалённые и оставшиеся legacy-стили

Из рабочих компонентов главной и overview удалено использование:

- тёмного CommandCenter;
- старого shield-логотипа;
- route-specific violet/gradient presentation;
- старых overview cards, badges и quality blocks;
- дублирующих домашних графиков.

В `globals.css` пока остаётся неиспользуемый legacy-блок старых `.overview-*` selectors как migration compatibility. Новый `/overview` использует V2 utility classes и только узкие новые selectors карты/печати. Удаление всего legacy-блока безопаснее выполнить в отдельной cleanup-фазе после миграции оставшихся expert pages.

## Проверки

Проверены:

- 154 frontend-теста;
- ESLint;
- TypeScript strict;
- production build;
- 1440, 1280, 768 и 375 px;
- три слоя карты;
- регион с данными и без данных;
- неизвестный регион и Шымкент;
- mobile ranked list и раскрытая карта;
- timeline, empty/error/insufficient;
- loading/error/empty главной;
- print media;
- Fullscreen API и fallback;
- reduced motion;
- эквивалент zoom 200%;
- отсутствие document overflow;
- отсутствие ошибок браузерной консоли.

Analytics visual harness создаёт 29 снимков во временном каталоге и не добавляет бинарные файлы в репозиторий.
