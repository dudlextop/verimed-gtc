# Реализация Verimed V2: review flow

## Границы фазы

Фаза `design-v2-review-flow` мигрирует только очередь `/signals`, быстрый просмотр и карточку `/signals/[id]`. Backend, аналитические формулы, маршруты, серверная пагинация, URL-параметры и экспертные API не изменялись. Страницы организаций, моделей, сводной аналитики, обзора и экспертной работы сохраняют прежнее содержание.

## Очередь сигналов

- Заголовок, показатели, фильтры, таблица, мобильные карточки и состояния собраны на foundation V2.
- Представления «Все сигналы», «Приоритетные», «Без решения», «На рассмотрении» и «Завершённые» продолжают задаваться URL.
- Поиск, основные и дополнительные фильтры используют существующие query parameters. Отдельного клиентского набора фильтров нет.
- Сортировка и пагинация остаются серверными. В таблице индикатор сортировки есть только у поддерживаемых колонок.
- Desktop-строка открывает быстрый просмотр; checkbox и overflow-меню останавливают событие строки.
- На ширине до 768 px таблица заменяется `MobileObjectCard`, горизонтальная прокрутка документа не используется.

## Выбор и экспорт

Выбор хранится в `useScopedSelection`. Его signature включает фильтры, поиск и сортировку, но не страницу. Поэтому IDs сохраняются между страницами одной выборки и очищаются при изменении условий с коротким уведомлением.

- `GET /api/exports/signals.csv` получает текущие filters и sort без ограничения текущей страницей.
- `POST /api/exports/signals.csv` получает только явно выбранные IDs.
- Экспорт всей выборки находится во вторичной зоне заголовка; на мобильном — в `OverflowActions`.
- Экспорт выбранных появляется только при непустом выборе.
- Ошибка и ограничение размера не очищают selection и не меняют URL.
- Имя файла извлекается из `Content-Disposition`; loading, success и error обрабатывает общий download client.

## Быстрый просмотр

`SignalPreviewPanel` использует белую правую панель шириной 30 rem на desktop и полноэкранный sheet на меньших ширинах. Первый экран содержит приоритет, статус, услугу, организацию, финансовую значимость, основную причину и одно основное действие. Обоснование, связанные записи, ограничения и последнее решение закрыты по умолчанию.

Панель сохраняет очередь текущей страницы, предыдущий/следующий сигнал, переход в полную карточку и обновление статуса строки. Реализованы focus trap, Escape, возврат фокуса, блокировка прокрутки background и touch targets 44×44 px.

## Карточка сигнала

Карточка сохраняет четыре этапа:

1. Сводка.
2. Обоснование.
3. Связи.
4. Решение.

Desktop и mobile используют один горизонтальный selector. На мобильном названия сокращены до «Сводка», «Обоснование», «Связи», «Решение» и прокручиваются без сжатия. Активный этап определяется наблюдением секций, отмечается `aria-current="step"`, поддерживает стрелки, Home и End и прокручивается в видимую область с учётом `prefers-reduced-motion`.

Сводка не повторяет показатели заголовка. Обоснование сначала показывает три основных фактора в едином формате. Связанные записи представлены временной последовательностью, длинные и вторичные блоки раскрываются по запросу. Решение сохраняет комментарии, feedback, последние события, полную историю и историю повторного появления.

## Действия

`StickyActionBar` содержит одно основное действие — «Направить на углублённую проверку». Подтверждение, отклонение и запрос сведений находятся в рабочем `OverflowActions`; отклонение не оформлено как удаление данных. После сохранения основным действием становится переход к следующему сигналу. Нижний отступ и safe area не дают панели перекрыть содержание.

## Сохранение контекста

- `returnTo` хранит URL очереди с фильтрами, сортировкой и страницей.
- `queueIds` задаёт соседние сигналы текущей выборки.
- Перед переходом сохраняется scroll position очереди.
- Возврат восстанавливает URL и прокрутку.
- Решение обновляет только текущий объект; переход на главную не выполняется.

## Accessibility

Проверены keyboard-only открытие строки, независимый checkbox, сортировка, вкладки, disclosures, focus trap панели и диалога, Escape, return focus, предыдущий/следующий сигнал, selector этапов и действия. Состояния сообщаются через live region, выбранная строка различима не только цветом, touch targets на mobile не меньше 44 px. Анимации используют перечисленные свойства и отключаются при reduced motion.

## Visual regression

`npm run visual:review-flow` использует Chrome DevTools Protocol, ждёт фактические API-состояния и задаёт точные viewport через device emulation. Матрица содержит 45 временных снимков: default, filters, selection, empty, loading и error списка; быстрый просмотр, disclosure, loading и error панели; четыре этапа карточки и reduced motion — на 1440, 768 и 375 px. Бинарные файлы не включаются в репозиторий.

## Legacy

Legacy-стили страниц организаций, моделей, главной, overview, reviews, журнала, методики и профиля сохранены. Совместимые старые badge-компоненты продолжают использоваться только внутри немигрированных экранов и отдельных связанных данных. Следующая page-level фаза может мигрировать организации без изменения review flow.
