# Expert pages V2 — реализация

## Объём фазы

Фаза `design-v2-expert` переносит на V2 только:

- результаты экспертной оценки `/reviews`;
- журнал решений `/decision-journal`;
- методику анализа `/methodology`;
- локальный профиль `/profile`;
- форму редактирования разрешённых полей локального профиля;
- visual regression и accessibility-проверки этих экранов.

Backend, API, аналитические формулы, база данных, миграции и уже завершённые продуктовые сценарии не изменялись.

## Результаты экспертной оценки

`/reviews` использует `PageHeader`, `MetricStrip`, `DataPanel`, `EmptyState` и общую V2-тему Recharts.

Первый уровень показывает:

- общее число зафиксированных решений;
- долю подтверждений;
- долю неподтверждённых сигналов;
- число сигналов без решения.

При достаточной выборке отображаются две визуализации: распределение решений и оценка полезности сигналов. Они используют только существующие агрегаты API, отключают декоративную анимацию и имеют текстовую альтернативу. Разбивки по типам сигналов и моделей остаются компактными рабочими списками.

При недостаточной выборке графики не создаются. Экран показывает одно `insufficient`-состояние с фактическим сообщением API, сохраняя единственную основную CTA к журналу решений.

## Журнал решений

`/decision-journal` использует URL как источник фильтров и страницы. Первые три условия — поиск, тип объекта и решение. Остальные условия раскрываются через `FilterBar`; активные условия можно удалить по одному или сбросить целиком.

Список использует:

- `DataTableShell` на desktop;
- компактные event cards на mobile;
- серверную пагинацию;
- спокойное состояние целостности;
- строку/карточку как основное действие открытия события.

Панель события показывает только рабочий контекст: действие, объект, решение, специалиста, основание, комментарий и дату. Hash, fingerprint, технический payload и внутренние пути не отображаются. Панель блокирует прокрутку страницы, закрывается по Escape, удерживает фокус и возвращает его исходной строке.

## Методика анализа

`/methodology` сохраняет полный API-контент, но организует его как четыре последовательных этапа:

1. подготовка и сопоставление данных;
2. выявление проверяемых отклонений;
3. формирование оценки;
4. экспертная проверка и решение.

Описание этапа видно сразу, подробности раскрываются доступным `details/summary`. Качество текущего анализа вынесено в отдельный компактный блок. Юридически аккуратное ограничение показывается один раз: сигнал является основанием для проверки, а финальный вывод делает специалист.

Новые алгоритмы, формулы и показатели не добавлялись.

## Локальный профиль

`/profile` использует существующий versioned local profile store. Первый уровень содержит нейтральный avatar/initials, отображаемое имя, должность, подразделение и контактную почту. Фиктивная активность, права, пароль, безопасность, logout и другие account-функции отсутствуют.

Форма редактирует только:

- `displayName`;
- `jobTitle`;
- `department`;
- `contactEmail`;
- `initials`;
- `neutralAvatarPreset`.

Поля имеют labels, autocomplete, ограничения длины, inline validation и фокус на первой ошибке. Пробелы нормализуются существующим валидатором store. Сохранение обновляет summary и Sidebar без reload; reset восстанавливает synthetic fallback.

Для синхронизации текущей вкладки store отправляет локальное событие `verimed:local-profile-change`. Синхронизация между вкладками по стандартному `storage` event сохранена. При недоступном localStorage форма честно сообщает об ограничении и блокирует сохранение.

Локальный профиль не является учётной записью и не изменяет backend identity.

## Удалённые и оставшиеся legacy-стили

Из четырёх маршрутов удалено использование:

- violet/purple presentation;
- тёмной шапки таблицы журнала;
- route-specific gradients;
- одинаковых тяжёлых KPI-карточек;
- статичного профиля с декоративным portrait;
- пустых больших панелей при недостаточной выборке;
- технических полей event preview.

Legacy HSL-токены и совместимые стили остальных маршрутов остаются в `globals.css`. Их удаление относится к отдельной cleanup-фазе после завершения всех page-level миграций.

## Проверки

Проверены:

- frontend tests;
- ESLint;
- TypeScript strict;
- production build Next.js;
- 1440, 1280, 768 и 375 px;
- loading, empty/insufficient и error;
- фильтры и пагинация журнала;
- Escape, focus trap и return focus;
- disclosures методики;
- validation, save, reload persistence, reset и storage sync профиля;
- недоступный localStorage;
- reduced motion;
- эквивалент zoom 200%;
- отсутствие горизонтального переполнения;
- отсутствие ошибок браузерной консоли.

Команда `npm run visual:expert` создаёт 45 снимков во временном каталоге. Бинарные файлы в репозиторий не добавляются.
