# CSV-экспорт

## Назначение

Серверный экспорт подготавливает фактическую отфильтрованную выборку, а не только текущую страницу. На `/signals` он интегрирован как вторичное действие для текущей выборки и как контекстное действие для выбранных сигналов. Экспорт организаций остаётся подготовленным для следующей page-level миграции.

## API

### Сигналы текущей выборки

`GET /api/exports/signals.csv`

Поддерживаются те же условия, что у списка сигналов: `search`, `level`, повторяемый `levels`, `organization_id`, `region`, `anomaly_type`, `status`, `priority_level`, `financial_min`, `financial_max`, `has_decision`, `period_months`, `date_from`, `date_to`, `sort`, `direction`. Параметры `page` и `page_size`, если переданы, не ограничивают файл.

### Выбранные сигналы

`POST /api/exports/signals.csv`

```json
{
  "signal_ids": [12, 25, 41]
}
```

IDs — положительные целые числа. Повторы удаляются с сохранением первого порядка. Пустой список и некорректный тип возвращают `422`; отсутствующий сигнал — структурированный `404` с `missing_ids`.

### Медицинские организации

`GET /api/exports/organizations.csv`

Поддерживаются `search`, `region`, `organization_type`, `risk_level`, `status`, `sort`, `direction`. Файл содержит всю отфильтрованную выборку.

## Колонки

Сигналы:

- сигнал;
- дата услуги;
- медицинская организация и регион;
- медицинская услуга;
- тип и основная причина отклонения;
- оценка и уровень риска;
- приоритет и уровень приоритета;
- финансовая значимость;
- статус экспертной оценки.

Организации:

- медицинская организация, регион и тип;
- количество услуг и сигналов;
- оценка и уровень риска;
- приоритет и уровень приоритета;
- финансовая значимость;
- статус экспертной оценки.

В файл не включаются fingerprint, hash, внутренние служебные поля и сведения о пациенте.

## Формат и безопасность

- UTF-8 с BOM и разделителем `;` для корректного открытия в Excel;
- даты — `ДД.ММ.ГГГГ`, суммы — с пробелами и знаком тенге;
- кавычки и переносы экранируются стандартным CSV writer;
- текст, начинающийся с `=`, `+`, `-`, `@`, табуляции или carriage return, получает безопасный апостроф;
- отрицательные числовые значения не изменяются, потому что защита применяется по типу поля;
- `Content-Disposition` содержит безопасное ASCII-имя файла;
- ответы не кэшируются.

Максимум — 5 000 строк для отфильтрованной выборки и 1 000 выбранных сигналов. При превышении API возвращает `413 export_limit_exceeded` и предлагает сузить фильтры; молчаливого обрезания нет.

## Frontend-клиент

`frontend/lib/download.ts` поддерживает GET, POST, имя из `Content-Disposition`, fallback-имя, JSON-ошибки, `AbortSignal` и обязательное освобождение object URL. `useFileDownload` отменяет активный запрос при размонтировании. `ExportAction` остаётся независимым от конкретного endpoint и принимает callback и описание области, например «Текущая выборка» или «Выбрано: 7».

На `/signals` GET-запрос строится из текущих URL-фильтров и сортировки без `page` и `page_size`. POST получает IDs из `useScopedSelection`. Selection сохраняется между страницами только при неизменном filter signature; ошибка или лимит экспорта не очищают выбранные строки.
