# Региональный мониторинг

## Контракт overview

`GET /api/analytics/overview` имеет `schema_version: 2` и аддитивно возвращает:

- `timeline` — тот же упорядоченный ряд, что `GET /api/analytics/timeline`;
- `regional_monitoring` — только регионы, для которых существуют фактические сигналы.

Элемент регионального мониторинга:

| Поле | Значение |
|---|---|
| `region_name` | Каноническое русское название |
| `region_code` | Стабильный код региона |
| `signal_count` | Число сущностей «сигнал» |
| `unique_record_count` | Число уникальных медицинских записей в финансовом snapshot |
| `financial_significance` | Сумма уникальных записей, связанных с сигналами |
| `organization_count` | Число организаций с сигналами |
| `maximum_priority` | Максимальный существующий приоритет сигнала |
| `leading_organization` | Организация с максимальным существующим приоритетом |

`signal_count` и `unique_record_count` не взаимозаменяемы: один сигнал может учитывать несколько связанных записей, а одна запись не должна повторно увеличивать финансовую сумму.

Финансовая значимость и уникальные записи берутся из последнего `FinancialImpactSnapshot`; приоритеты — из существующих `ReviewPriority` и `OrganizationPrioritySnapshot`. Новых формул нет. Timeline переиспользует `get_timeline`, поэтому второго расчёта динамики не существует.

## Канонизация

`backend/app/services/regions.py` применяет только явные алиасы: нормализует Unicode, регистр и повторные пробелы, но не делает fuzzy matching. Разные регионы не объединяются по похожему написанию.

Неизвестное название сохраняется отдельным агрегатом с детерминированным кодом `unknown-<hash>`. Такой код не получает случайную геометрию и не приводит к ошибке overview.

## Геометрия и mapping

Статический файл: `frontend/public/maps/kazakhstan-adm1.geojson` (475 485 байт).

Источник: [geoBoundaries KAZ ADM1](https://www.geoboundaries.org/api/current/gbOpen/KAZ/ADM1/), commit `9469f09`, исходные границы OpenStreetMap / Wambacher, лицензия Open Data Commons Open Database License 1.0. Требуемая атрибуция сохранена здесь и рядом с asset.

Typed mapping находится в `frontend/lib/region-map.ts` и связывает:

- API region code;
- каноническое русское название;
- `shapeID` геометрии;
- доступную подпись.

Отсутствующий агрегат остаётся `undefined`, а не превращается в ноль. Неизвестный API code попадает в диагностический список и не окрашивает область.

Ограничение источника: опубликованная геометрия отражает административное деление 2017 года и содержит 16 объектов. Для Шымкента используется стабильный API-код `KZ-SHY`, но намеренно не назначается геометрия, поскольку в источнике нет отдельного объекта города. Это безопаснее ошибочного сопоставления с Туркестанской областью.

## Представление в аналитическом обзоре

`/overview` использует одну интерактивную SVG-карту и три слоя:

- сигналы;
- финансовая значимость;
- максимальный приоритет.

Интенсивность рассчитывается только внутри фактических агрегатов выбранного слоя. Регион без агрегата получает состояние `Нет данных` и не входит в сравнительную шкалу.

Основная активация региона ведёт в `/organizations?region=<каноническое название>`. В tooltip показываются только поля текущего API: сигналы, уникальные записи, финансовая значимость, организации, максимальный приоритет и ведущая организация.

На 375 px главным представлением является ранжированный список. Карта доступна вторым уровнем через disclosure. Неизвестные API-коды и Шымкент остаются в списке, но не получают случайную геометрию.

Карта имеет текстовую альтернативу, доступные названия областей, клавиатурную активацию и видимый фокус. Ошибка загрузки GeoJSON является частичной: остальной overview и региональный список продолжают работать.
