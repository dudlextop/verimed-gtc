# Проверка целостности истории решений

Каждая цепочка событий строится отдельно для сочетания типа объекта и fingerprint. `previous_event_hash` указывает на контрольную сумму предыдущего события. `event_hash` — SHA-256 от fingerprint, типа объекта, действия, решения, причины, комментария, автора, времени и предыдущей контрольной суммы.

Сервис проверки читает события по порядку, пересчитывает суммы, проверяет ссылки и возвращает число проверенных событий и найденных несоответствий. Проверка не исправляет данные автоматически. Результат сохраняется в `DecisionJournalIntegrityCheck` и в интерфейсе отображается как компактная отметка.

Hash-цепочка помогает обнаружить незаметное изменение локальной истории. Она не является внешней криптографической гарантией, электронной подписью, юридически значимым документооборотом или распределённым реестром. Администратор базы с полным доступом теоретически может изменить события и пересчитать всю цепочку; для более сильной гарантии нужен независимый внешний якорь, который не входит в текущий этап.
