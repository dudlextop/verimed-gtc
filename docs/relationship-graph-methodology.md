# Методика графа связей

## Рабочий вопрос

Граф помогает специалисту понять, какие организации, врачи, обезличенные пациенты, услуги и сигналы образуют выбранную повторяющуюся модель.

## Узлы

- повторяющаяся модель;
- медицинская организация;
- врач с обезличенным кодом;
- пациент с обезличенным кодом;
- медицинская услуга;
- сигнал риска.

Размер узла зависит от числа связанных сигналов; для организаций, услуг и сигналов дополнительно возвращается финансовая значимость. Настоящие персональные данные не используются.

## Связи

- модель включает сигнал;
- сигнал относится к организации;
- сигнал связан с врачом;
- сигнал связан с пациентом;
- сигнал относится к услуге;
- врач оказывает услугу;
- пациент получил услугу.

API возвращает структурированные `nodes` и `edges`, а не готовое изображение. Каждая связь содержит тип и вес.

## Ограничение сложности

По умолчанию API возвращает до 60 наиболее значимых узлов, интерфейс сначала показывает до 30. Приоритет имеют модель, основные организации, услуги и врачи. Специалист может раскрыть дополнительные связи, скрыть типы узлов и центрировать граф на выбранном объекте.

Переходы из узлов организации и сигнала ведут в существующие карточки. Узлы врача, услуги и пациента центрируют граф, так как отдельных карточек для этих сущностей на текущем этапе нет.

## Читаемость и доступность

- фильтры и узлы реализованы кнопками и ссылками;
- доступна клавиатурная навигация и видимый фокус;
- цвет дополняется подписью типа узла;
- подробность выбранного узла повторяется текстом под графом;
- широкая рабочая область прокручивается горизонтально на планшете.
