# Методика стабильного fingerprint сигнала

## Назначение

Fingerprint связывает одно логическое отклонение между последовательными запусками анализа. Текущий идентификатор сигнала может измениться, а экспертная история остаётся доступной по 64-символьному SHA-256 fingerprint.

## Канонизация

Версия структуры `1` включает тип сигнала, организацию, обезличенного пациента, врача, медицинскую услугу, нормализованный период, тип связи, число связанных записей и диапазон суммы. Для частотных и месячных отклонений период нормализуется до месяца, для коротких интервалов — до недели, для остальных правил — до даты. Сумма округляется до устойчивого диапазона 500 или 2 500 тенге.

Структура сериализуется в JSON с сортировкой ключей, фиксированными разделителями и UTF-8, затем хешируется SHA-256. Идентификатор запуска, идентификатор сигнала и время создания в структуру не входят. В открытом виде fingerprint не содержит код пациента или врача.

Существенное изменение услуги, участников, временного контекста, типа связи или диапазона суммы создаёт новый fingerprint. Несколько текущих записей, описывающих одно групповое отклонение, могут иметь общий fingerprint; снимок появления хранит один репрезентативный сигнал на запуск.

## Ограничения

Канонизация является прикладной эвристикой. Изменение входных справочников или версии структуры требует отдельной стратегии миграции. Полное пересоздание синтетического набора начинает новый контекст данных.
