# Визуальная регрессия

## Автоматический сценарий

Сценарий `frontend/scripts/capture-visuals.mjs` использует установленный Chrome или Chromium и создаёт снимки вне репозитория. Он автоматически получает идентификаторы приоритетного сигнала, организации и модели через существующий API.

Перед запуском должны работать backend и production-сборка frontend. Это исключает попадание состояния компиляции маршрута в снимок:

```bash
cd frontend
NEXT_PUBLIC_API_URL=http://localhost:8000/api npm run build
npm start
# во втором терминале
npm run visual:check
```

По умолчанию снимки сохраняются в `/tmp/verimed-visuals`. Каталог можно изменить:

```bash
VERIMED_VISUAL_OUTPUT=/tmp/verimed-review npm run visual:check
```

Если Chrome установлен нестандартно, укажите `CHROME_BIN`. Бинарные снимки в репозиторий не добавляются. `manifest.json` фиксирует маршрут, ширину и имя каждого файла.

## Foundation V2

Внутренний маршрут `/foundation-preview` не входит в стандартную пользовательскую матрицу. Для проверки foundation запустите frontend с явным флагом и используйте отдельную команду:

```bash
cd frontend
NEXT_PUBLIC_ENABLE_FOUNDATION_HARNESS=true NEXT_PUBLIC_API_URL=http://localhost:8000/api npm run dev
# во втором терминале
VERIMED_VISUAL_OUTPUT=/tmp/verimed-foundation npm run visual:foundation
```

Команда создаёт стандартные 27 снимков и ещё три снимка foundation на 1440, 768 и 375 px. В production маршрут возвращает 404 без `NEXT_PUBLIC_ENABLE_FOUNDATION_HARNESS=true`; включать его в пользовательское развёртывание не требуется.

15 июля 2026 года foundation-матрица успешно создала 30 снимков. Дополнительно проверены 200% device scale, `prefers-reduced-motion`, мобильные touch targets, отсутствие горизонтального переполнения и ошибок браузерной консоли.

## Shell V2

Для оболочки существует отдельная матрица из пяти маршрутов и четырёх ширин:

```bash
cd frontend
VERIMED_VISUAL_OUTPUT=/tmp/verimed-v2-shell npm run visual:shell
```

Маршруты: `/`, `/signals`, `/organizations`, `/patterns`, `/profile`. Ширины: 1440, 1280, 768 и 375 px. Команда создаёт 20 закрытых состояний и не включает `/overview`, потому что обзор использует отдельную оболочку.

Дополнительная интерактивная проверка выполняется для sheet:

- 768 px: закрытое и открытое меню;
- 375 px: закрытое и открытое меню;
- Escape, focus trap и возврат фокуса;
- блокировка прокрутки фона;
- 200% zoom и `prefers-reduced-motion`;
- `scrollWidth === innerWidth` на рабочих маршрутах.

15 июля 2026 года production-матрица создала 20 маршрутных снимков и 5 интерактивных снимков shell. Проверены реальные API-данные, открытые и закрытые состояния sheet, эквивалент 200% zoom, Escape, focus trap, return focus и отсутствие document overflow.

## Review flow V2

Для очереди и карточки сигнала используется отдельная команда:

```bash
cd frontend
VERIMED_VISUAL_OUTPUT=/tmp/verimed-v2-review-flow npm run visual:review-flow
```

Сценарий управляет headless Chrome через DevTools Protocol: ждёт фактическое состояние API, задаёт точный CSS viewport и не фиксирует случайный skeleton из-за virtual time. Это необходимо для корректной проверки 375 px в Chrome, где простого `--window-size` недостаточно.

Матрица содержит 45 снимков на 1440, 768 и 375 px:

- очередь по умолчанию;
- активные фильтры;
- активный выбор;
- empty, loading и error;
- открытый быстрый просмотр, вторичное disclosure, loading и error;
- четыре этапа карточки сигнала;
- карточка при `prefers-reduced-motion: reduce`.

16 июля 2026 года матрица успешно создана во временном каталоге. Дополнительно в браузере проверены 1280 и 1024 px, отсутствие document overflow, полноэкранная мобильная панель, 44×44 px controls, Escape, focus trap, return focus, keyboard selector этапов, sticky bar, reduced motion и масштабирование 200%. Бинарные снимки в репозиторий не добавлены.

## Медицинские организации V2

Для списка и карточки организации используется отдельная команда:

```bash
cd frontend
VERIMED_VISUAL_OUTPUT=/tmp/verimed-v2-organizations npm run visual:organizations
```

Матрица включает 1440, 1280, 768 и 375 px там, где ширина является ключевой, и целевые снимки состояний:

- список по умолчанию и с активными фильтрами;
- loading, empty и error списка;
- loading и error экспорта;
- первый экран карточки организации;
- сравнение с сопоставимой группой;
- связанные сигналы и модели;
- длинное название организации;
- пустые связанные сигналы и модели;
- loading и error карточки.

Сценарий использует фактические API-данные. Для воспроизводимых edge states он временно подменяет только соответствующий HTTP-ответ внутри отдельного headless Chrome; продуктовый код и snapshot данных не изменяются. Бинарные снимки сохраняются только во временном каталоге.

16 июля 2026 года матрица успешно создала 36 снимков. В неё включены отдельные состояния `prefers-reduced-motion: reduce` и масштаба 200%. Для каждого состояния автоматически проверены отсутствие горизонтального переполнения документа, необработанных исключений и ошибок браузерной консоли.

## Analytics V2

Для главной и полноэкранного аналитического обзора используется отдельная команда:

```bash

VERIMED_VISUAL_OUTPUT=/tmp/verimed-v2-analytics npm run visual:analytics
```

Матрица создаёт 29 снимков и проверяет:

- главную на 1440, 1280, 768 и 375 px;
- loading, empty и error главной;
- overview на 1440, 1280, 768 и 375 px;
- слои карты `Сигналы`, `Финансовая значимость` и `Приоритет`;
- выбранный регион и регион без агрегата;
- основной mobile ranked list и раскрытую карту;
- insufficient и error состояния timeline;
- частичную ошибку GeoJSON;
- print media;
- fullscreen;
- reduced motion;
- эквивалент zoom 200%.

Сценарий автоматически проверяет горизонтальное переполнение и ошибки браузерной консоли. При обнаружении overflow он выводит список элементов, вышедших за viewport. Такой check выявил visually-hidden table текстовой альтернативы timeline, которая расширяла документ на 375 px; после замены на скрытый текст повторная матрица прошла.

16 июля 2026 года analytics-матрица успешно создана во временном каталоге. Карта использовала фактический `regional_monitoring`, timeline — фактический overview payload. Бинарные снимки в репозиторий не добавлены.

## Финальная QA-матрица

Команда:

```bash
cd frontend
VERIMED_VISUAL_OUTPUT=/tmp/verimed-v2-final-qa npm run visual:qa
```

Проверяет 12 маршрутов на 1440, 1280, 1024, 768, 375 и 320 px, а также отдельные состояния:

- быстрый просмотр сигнала и события журнала;
- выбранный узел relationship graph;
- слой приоритета региональной карты;
- открытый navigation sheet на 768 и 375 px;
- 200% zoom главной и карточки сигнала;
- reduced motion overview;
- короткий mobile viewport и landscape;
- стандартную печать `/overview`.

Каждый browser run проверяет:

- ожидаемое состояние страницы;
- ошибки консоли и необработанные исключения;
- horizontal document overflow;
- повторяющиеся `id`;
- изображения без `alt`;
- controls без доступного имени;
- focusable elements внутри `aria-hidden`;
- touch targets на mobile;
- компактную performance-сводку.

В `manifest.json` сохраняются результаты аудита и runtime-метрики для каждого снимка.

16 июля 2026 года выполнены:

- master-набор — 36 снимков;
- расширенный cross-route QA — 86 снимков;
- review flow — 45 снимков;
- organizations — 36 снимков;
- patterns — 43 снимка;
- analytics — 29 снимков;
- expert pages — 45 снимков.

Всего создано 320 актуальных снимков. Во всех manifests — 0 runtime-a11y/overflow/console issues. Стандартная печать Chrome сформировала пять альбомных страниц; интерактивные controls скрыты, логотип, период, карта, timeline и текстовые альтернативы сохранены.

В ходе QA исправлены сжатие mobile-кнопки удаления длинного filter chip, семантика navigation overlay и контраст `text-muted`. После исправлений соответствующие матрицы выполнены повторно. Бинарные результаты остались во временных каталогах и в репозиторий не включены.

Playwright и axe не являются зависимостями проекта. Их роль выполняет существующий headless Chrome/CDP harness с behavior assertions и runtime accessibility audit; добавление параллельного тяжёлого стека в cleanup-фазе не потребовалось.
