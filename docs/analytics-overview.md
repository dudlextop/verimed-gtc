# Аналитический обзор Verimed

## Назначение и аудитория

Маршрут `/overview` предназначен для управленческого представления результатов анализа на совещании, при защите продукта и при записи короткого обзора. Это экран действующего продукта: показатели ведут к организациям, сигналам, повторяющимся моделям и приоритетной очереди. Он не заменяет сводную аналитику и рабочие карточки специалиста.

Стандартная боковая навигация на маршруте не отображается. Вход выполняется явным действием «Открыть аналитический обзор» на главной странице. Рабочая структура основной навигации не меняется.

## Структура

1. Верхняя панель: Verimed, период, время анализа, количество организаций и актуальность данных.
2. Главный результат: объём обработанных услуг, доля и число отобранных записей, сокращение ручного просмотра и потенциальный объём проверки.
3. Три объекта внимания: организация, сигнал и повторяющаяся модель.
4. Системные выводы: детерминированные короткие формулировки из структурированных агрегатов.
5. Качество анализа: точность, полнота, F1-мера и доля ложноположительных записей.
6. Повторяющиеся модели: сводка и распределение по типам.
7. Изменения между запусками и одно рекомендуемое действие.

На широком экране блоки образуют компактную управленческую сетку. На планшете и телефоне сетка становится последовательной; горизонтальная прокрутка не используется.

## Источники данных

Новый backend-маршрут не добавлялся. Экран переиспользует:

- `GET /api/analytics/summary`;
- `GET /api/analytics/command-center`;
- `GET /api/analytics/changes`;
- `GET /api/analytics/financial-impact`;
- `GET /api/analytics/priority-summary`;
- `GET /api/analytics/pattern-summary`;
- `GET /api/analytics/pattern-changes`;
- `GET /api/patterns` для распределения существующих моделей по типам;
- `GET /api/analysis/metrics`;
- `GET /api/analytics/expert-review-summary`.

Страница не пересчитывает риск, приоритет, финансовую значимость или качество анализа. Точное количество отобранных записей определяется как сумма истинно- и ложноположительных записей из `GET /api/analysis/metrics`; доля отбора отображается отдельно и не используется для обратного расчёта количества после округления. Клиентская логика только форматирует значения и группирует полученный список моделей для компактной диаграммы.

Количество отобранных записей не равно финансовому охвату. В финансовый охват входят уникальные основные и связанные записи сигналов; поэтому он может быть больше числа сигналов. Определения собраны в [key-metrics-definitions.md](key-metrics-definitions.md).

## Состояния

До завершения всех запросов показывается единый скелетон. При ошибке выводится одно состояние с повторной попыткой. Предусмотрены отдельные тексты для отсутствия прошлого запуска, моделей и достаточной выборки экспертных решений. Анализ старше семи суток отмечается как требующий обновления.

## Полноэкранное поведение

Действие «Развернуть обзор» вызывает стандартный Fullscreen API только после явного нажатия. Стандартный выход браузера, включая Escape, сохраняется. При отсутствии или ошибке API экран продолжает работать и показывает краткое пояснение; отдельная настройка режима не хранится.

## Печать

Печать запускается стандартным диалогом браузера. Стили `@media print`:

- скрывают переходы, печатную кнопку и полноэкранное действие;
- переводят поверхности на светлый фон;
- сохраняют главные показатели, выводы и подпись о синтетических данных;
- не разрывают ключевые карточки по возможности браузера;
- используют альбомную ориентацию без собственного генератора PDF.

## Ограничения

- метрики качества рассчитаны на воспроизводимом синтетическом наборе с известной эталонной разметкой;
- системные выводы детерминированы и не используют генеративную модель;
- полноэкранный режим и разрывы страниц при печати зависят от возможностей браузера;
- обзор показывает текущий управленческий срез и не заменяет изучение обоснований в рабочих карточках.
