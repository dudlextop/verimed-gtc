# Правила компонентов Verimed

## Foundation V2 и совместимость

Новые компоненты импортируются из `@/components/foundation`. `@/components/ui` остаётся совместимым фасадом для текущих страниц до их поэтапной миграции. Новые компоненты используют только `--v2-*` токены; legacy HSL-токены не расширяются.

Утверждённый логотип выводится через `BrandLogo`. Его нельзя заменять иконкой, текстовым суррогатом или самостоятельно перерисованным знаком.

## Оболочка и навигация

- рабочие маршруты используют белый Sidebar шириной 264 px на `xl` и выше;
- на меньших ширинах используется компактный header и единый светлый navigation sheet;
- состав, порядок и подписи маршрутов задаются в одном компоненте и не расходятся между desktop и mobile;
- active route обозначается `aria-current`, левой линией, фоном и начертанием, а не только цветом;
- hover навигации не использует transform;
- sheet закрывается Escape и возвращает фокус trigger-элементу;
- profile trigger является прямой ссылкой на `/profile`; пустые account-меню не создаются;
- `/overview` сохраняет отдельную полноэкранную оболочку без рабочего Sidebar.

## Кнопки

- на рабочей поверхности одновременно используется одна основная кнопка;
- рядом допускаются не более двух вторичных действий;
- `destructive` применяется только к действительно необратимому действию и не используется для обычного отрицательного экспертного вывода;
- `outline` и `danger` временно поддерживаются как legacy-алиасы `secondary` и `destructive`;
- высота обычной кнопки — 44 px, compact — 40 px;
- кнопка без текста должна иметь `aria-label` и область не меньше 44×44 px;
- переходы перечисляют свойства явно; `transition-all` запрещён.

## Показатели и бейджи

- `DomainIndicator kind="priority"` — только приоритет сигнала или организации;
- `DomainIndicator kind="risk"` — только оценка риска;
- `DomainIndicator kind="importance"` — только важность повторяющейся модели;
- `DomainIndicator kind="stability"` — только устойчивость;
- `DomainIndicator kind="reviewStatus"` — состояние экспертной работы;
- совместимые `PriorityBadge`, `RiskBadge`, `ImportanceBadge`, `StabilityBadge`, `StatusBadge` используют `DomainIndicator` внутри;
- суммы оформляются через `FinancialValue` и tabular numerals, а не monospace;
- яркие семантические токены предназначены для маркеров и графиков; мелкий текст использует контрастные `*-text` токены.

## Таблицы и мобильные карточки

На настольном экране используется `DataTableShell` без вертикальных разделителей. Высота header — 48 px, строки — 64 px. Сортировка отображается только при наличии рабочего обработчика. Выбранная строка получает мягкий синий фон и левую линию. На ширине до 768 px списки переходят в `MobileObjectCard` с главным показателем, названием, контекстом, суммой, статусом и причиной.

## Формы

Поля используют `Input`, `Search`, `Select`, `Textarea` и `Checkbox` высотой не меньше 44 px. Видимая подпись обязательна для форм решений; в компактных фильтрах допустима подпись для экранного читателя и ясное значение по умолчанию. `FilterBar` показывает основные фильтры, счётчик активных условий, действие «Сбросить все» и доступное раскрытие дополнительных фильтров. Ошибка показывается рядом с формой, введённый комментарий сохраняется при неуспешном запросе.

## Состояния

- загрузка сохраняет геометрию через `PageSkeleton` варианта dashboard, list, detail, journal или overview;
- ошибка содержит понятное описание и действие «Повторить»;
- `EmptyState` использует один из вариантов empty, insufficient, error, history или stale, объясняет причину и содержит максимум одно действие;
- успешное действие использует короткое уведомление;
- технические тексты ответа сервиса пользователю не показываются.

## Карточки решений

Факторы имеют одинаковый порядок: тип, вывод, вклад, фактическое значение, значение сравнения, пояснение. Вклад визуально вторичен. Закреплённая зона решений отделяется границей и мягкой тенью, не перекрывает содержимое и учитывает безопасную область мобильного экрана.

## Очередь проверки V2

- `/signals` использует одну URL-модель для представления, фильтров, поиска, сортировки и страницы;
- selection может переживать только смену страницы при неизменном filter signature;
- checkbox и меню строки не открывают быстрый просмотр;
- overflow-trigger в строке показывается как одна icon button с доступным названием;
- экспорт всей выборки находится во вторичной зоне, экспорт выбранных появляется только после выбора;
- на mobile показатели очереди образуют компактную сетку 2×2, а таблица заменяется карточками;
- быстрый просмотр показывает основной контекст до disclosures и содержит одну primary CTA;
- selector этапов имеет четыре короткие подписи, `aria-current="step"` и горизонтальный scroll на mobile;
- `StickyActionBar` показывает одну primary CTA, а остальные решения переносит в рабочее меню.

## Медицинские организации V2

- `/organizations` использует URL как единственный источник поиска, фильтров, сортировки и страницы;
- `FilterBar` допускает `primaryClassName` только для адаптивной раскладки существующих controls; на 1280 px поля не должны сжиматься ради размещения действий в той же строке;
- показатели всей выборки и текущей страницы должны быть подписаны различимо;
- риск организации выводится через `DomainIndicator kind="risk"`, приоритет — через `kind="priority"`;
- экспорт всей выборки является вторичным действием и исключает `page` и `page_size`;
- переход в карточку передаёт `returnTo`, чтобы список восстанавливал контекст;
- на первом уровне карточки организации находятся четыре показателя и одна CTA к отфильтрованным сигналам;
- причины ограничиваются четырьмя ведущими значениями, остальные раскрываются;
- сравнение с группой показывает значение организации, медиану, типичный диапазон, отклонение и текстовую интерпретацию;
- связанные сигналы переиспользуют таблицу и мобильные карточки review flow, а модели сохраняют отдельную семантику важности и устойчивости.
- длинная карточка использует якорный selector без отдельного tab-state; текущий раздел обозначается `aria-current="location"`.

## Повторяющиеся модели V2

- `/patterns` использует URL как единственный источник фильтров, сортировки и страницы;
- поиск не имитируется на frontend, если соответствующий параметр не поддержан backend;
- важность модели выводится через `DomainIndicator kind="importance"` и доминирует над устойчивостью;
- устойчивость использует `kind="stability"` и объясняет повторяемость, а не опасность;
- длинная карточка состоит из пяти якорных секций без отдельного tab-state;
- сначала показывается текстовый состав участников, затем relationship graph;
- topology, node IDs, edge IDs и layout algorithm графа не меняются presentation-компонентом;
- на mobile текстовый список связей является основным, canvas создаётся только после явного раскрытия;
- выбранный узел и ближайшие связи усиливаются, остальные элементы ослабляются;
- неизвестный тип узла получает нейтральный fallback;
- динамика важности и устойчивости строится только по фактической recurrence history; отсутствующие точки не достраиваются;
- входящие сигналы переиспользуют review-flow таблицу, мобильные карточки и `SignalPreviewPanel`;
- после сохранения решения `StickyActionBar` предлагает следующую модель в текущей очереди, а остальные решения остаются в `OverflowActions`.

## Аналитический обзор

- обзор использует отдельную оболочку без рабочей боковой навигации;
- верхняя панель содержит только контекст анализа, возврат в систему, печать и необязательное полноэкранное действие;
- главный вывод всегда строится из API и не содержит жёстко заданных аналитических значений;
- кликабельная карточка объекта целиком является ссылкой и имеет видимый фокус;
- на экране существует одно главное действие — переход к проверке;
- метрики качества всегда сопровождаются подписью о синтетическом наборе;
- при печати элементы с классом `print:hidden` исключаются, а ключевые секции защищаются от разрыва;
- fallback Fullscreen API отображается как краткое состояние и не блокирует экран.
- карта и динамика являются единственными двумя ведущими визуализациями;
- карта использует только локальную GeoJSON-геометрию и typed region mapping без fuzzy matching;
- слой карты оформляется как `radiogroup`, поддерживает стрелки/Home/End и не создаёт отдельный page mode;
- регион без агрегата показывает `Нет данных`, а не ноль;
- на mobile ранжированный список является основным, карта раскрывается вторым уровнем;
- timeline сохраняет порядок API, не достраивает периоды и имеет текстовую альтернативу.

## Главная аналитика V2

- главная является рабочим стартом и не дублирует полный overview;
- светлый CommandCenter содержит одну primary CTA, одну рекомендацию и вторичный переход в обзор;
- первый уровень показывает сигнал, организацию и модель с различной доменной семантикой;
- карта и timeline на главной не размещаются;
- экспертный контур и качество анализа допускается убирать на второй уровень disclosure;
- переходы к очереди и приоритетной организации сохраняются при визуальной миграции.

## Экспертные страницы V2

- `/reviews` использует одну `MetricStrip` и не более двух фактических визуализаций;
- при недостаточной выборке графики не рисуются, а одно `EmptyState variant="insufficient"` объясняет ограничение;
- `/decision-journal` хранит filters/page в URL, использует `FilterBar`, `DataTableShell` и mobile event cards;
- integrity отображается спокойным `InlineNotice`; hash, fingerprint и технический payload пользователю не показываются;
- event preview является доступной правой панелью с Escape, focus trap, return focus и блокировкой body scroll;
- `/methodology` использует линейную четырёхэтапную композицию; API-тексты не заменяются придуманными формулами;
- юридическое ограничение методики показывается один раз;
- `/profile` использует только versioned local profile store и не имитирует backend account;
- форма профиля имеет labels, autocomplete, inline validation, фокус первой ошибки, success/error feedback и вторичный reset;
- при недоступном localStorage форма сохраняет synthetic fallback и блокирует локальные записи;
- profile trigger обновляется через общий store, а не отдельное состояние Sidebar.

## Проверка компонента

Перед слиянием проверяются клавиатура, видимый фокус, контраст, 375/768/1280 px, состояния загрузки/ошибки/пустого результата, отсутствие `transition-all` и работа `prefers-reduced-motion`.

Внутренний стенд `/foundation-preview` не входит в навигацию и закрыт в production без явного флага. Он используется для проверки всех примитивов до миграции рабочих страниц.
