# Правила компонентов Verimed

## Кнопки

- на рабочей поверхности одновременно используется одна основная кнопка;
- рядом допускаются не более двух вторичных действий;
- `danger` применяется только к действительно опасному действию и не используется для обычного отрицательного экспертного вывода;
- кнопка без текста должна иметь `aria-label` и область не меньше 40×40 px.

## Показатели и бейджи

- `PriorityBadge` — только приоритет сигнала или организации;
- `RiskBadge` — только оценка риска;
- `ImportanceBadge` — только важность повторяющейся модели;
- `StabilityBadge` — только устойчивость;
- `StatusBadge` — состояние экспертной работы;
- суммы оформляются через `FinancialValue` или нейтральное моноширинное число.

## Таблицы и мобильные карточки

На настольном экране используется единая таблица без вертикальных разделителей. Вся рабочая строка интерактивна. На ширине до 768 px списки переходят в карточки с главным показателем, названием, контекстом, суммой, статусом и причиной.

## Формы

Поля используют `Input` и `Select`. Видимая подпись обязательна для форм решений; в компактных фильтрах допустима подпись для экранного читателя и ясное значение по умолчанию. Ошибка показывается рядом с формой, введённый комментарий сохраняется при неуспешном запросе.

## Состояния

- загрузка сохраняет геометрию через скелетоны;
- ошибка содержит понятное описание и действие «Повторить»;
- отсутствие данных объясняет причину;
- успешное действие использует короткое уведомление;
- технические тексты ответа сервиса пользователю не показываются.

## Карточки решений

Факторы имеют одинаковый порядок: тип, вывод, вклад, фактическое значение, значение сравнения, пояснение. Вклад визуально вторичен. Закреплённая зона решений отделяется границей и мягкой тенью, не перекрывает содержимое и учитывает безопасную область мобильного экрана.

## Аналитический обзор

- обзор использует отдельную оболочку без рабочей боковой навигации;
- верхняя панель содержит только контекст анализа, два рабочих перехода, печать и необязательное полноэкранное действие;
- главный вывод всегда строится из API и не содержит жёстко заданных аналитических значений;
- кликабельная карточка объекта целиком является ссылкой и имеет видимый фокус;
- на экране существует одно главное действие — переход к приоритетной проверке;
- метрики качества всегда сопровождаются подписью о синтетическом наборе;
- при печати элементы с классом `print:hidden` исключаются, а ключевые секции защищаются от разрыва;
- fallback Fullscreen API отображается как краткое состояние и не блокирует экран.

## Проверка компонента

Перед слиянием проверяются клавиатура, видимый фокус, контраст, 375/768/1280 px, состояния загрузки/ошибки/пустого результата, отсутствие `transition-all` и работа `prefers-reduced-motion`.
