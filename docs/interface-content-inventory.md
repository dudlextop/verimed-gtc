# Инвентаризация пользовательского содержания Verimed

Дата актуализации: 13 июля 2026 года.

Документ фиксирует пользовательские тексты после структурного упрощения. Динамические названия организаций, услуг, сигналов и моделей поступают из API.

## Навигация

### Основной уровень

- Сводная аналитика
- Медицинские организации
- Проверка
- Повторяющиеся модели

### Группа «Экспертная работа»

- Результаты экспертной оценки
- Журнал решений

### Вторичный уровень

- Методика анализа
- Профиль специалиста

«Загрузка данных» и «Настройки» не показываются, потому что пользовательский сценарий сохранения для них не завершён.

## Заголовки экранов и разделов

| Маршрут | Основные заголовки |
|---|---|
| `/` | Контроль медицинских услуг; Требуют внимания; Что изменилось; Приоритетные медицинские организации; Повторяющиеся модели, требующие внимания; Подробная аналитика |
| `/organizations` | Медицинские организации |
| `/organizations/{id}` | название организации; Почему организация имеет такой приоритет; Сравнение с сопоставимыми организациями; Повторяющиеся модели организации; Дополнительная аналитика; Последние сигналы |
| `/signals` | Проверка; Все сигналы; Приоритетные; Без решения; На рассмотрении; Завершённые |
| `/signals/{id}` | Сводка; Обоснование; Связанные записи; Решение специалиста; Показать все факторы; Перейти к следующему сигналу; Ограничения анализа; История повторного появления |
| `/patterns` | Повторяющиеся модели |
| `/patterns/{id}` | Сводка; Почему модель важна; Связи; Что формирует модель; Сигналы; Экспертная оценка; Что изменилось с предыдущего появления |
| `/reviews` | Результаты экспертной оценки |
| `/decision-journal` | Журнал решений |
| `/methodology` | Методика анализа |

## Иерархия показателей

| Объект | Главный показатель | Вторичные показатели |
|---|---|---|
| Сигнал | Приоритет проверки | Оценка риска; Финансовая значимость |
| Медицинская организация | Приоритет проверки | Оценка риска; Финансовая значимость; Количество сигналов |
| Повторяющаяся модель | Важность модели | Устойчивость; Финансовая значимость; Количество сигналов |

Финансовая значимость описывает масштаб услуг. В карточке объекта используется компактное пояснение: «Финансовая значимость не означает подтверждённое нарушение.»

## Действия

### Сигнал

- Основное: Направить на углублённую проверку.
- Вторичные: Подтвердить сигнал; Отклонить сигнал.
- Дополнительные: Начать проверку; Требуются дополнительные сведения; Добавить комментарий; Открыть полную историю решений; Открыть связанные объекты.
- После решения: Перейти к следующему сигналу; Вернуться в Проверку.

### Быстрый просмотр

- Основное: Начать проверку или Продолжить проверку.
- Навигация: Предыдущий сигнал; Следующий сигнал.
- Вторичное: Открыть карточку.
- Раскрываемые разделы: Обоснование; Связанные записи; Ограничения; Последнее экспертное решение.

### Повторяющаяся модель

- Основное: Направить на углублённую проверку.
- Вторичные: Подтвердить значимость; Признать несущественной.
- Дополнительные: Требуются дополнительные сведения; Добавить комментарий; Открыть полную историю решений.

Дополнительные действия собраны в меню «Другие действия». В карточке существует одна закреплённая панель решений.

## Статусы

### Сигнал

- Не проверено
- На рассмотрении
- Подтверждён сигнал
- Сигнал не подтверждён
- Требуются дополнительные сведения
- Направлено на углублённую проверку
- Проверка завершена

### Модель

- Не оценено
- На рассмотрении
- Значимость подтверждена
- Отмечено как несущественное
- Требуются дополнительные сведения
- Направлено на углублённую проверку
- Оценка завершена

### Уровни

- Риск и приоритет: Низкий; Средний; Высокий; Критический.
- Устойчивость: Низкая; Средняя; Высокая; Очень высокая.
- Важность: Низкая; Средняя; Высокая; Критическая.

## Таблицы

### Проверка

Обязательные заголовки: Приоритет; Медицинская организация; Медицинская услуга; Финансовая значимость; Основная причина; Статус; Дата. Оценка риска, пациент, тип сигнала и дополнительные факторы находятся в быстром просмотре.

### Повторяющиеся модели

Обязательные заголовки: Важность; Название модели; Тип; Организация; Финансовая значимость; Устойчивость; Статус. Сортировка выполняется отдельным подписанным списком.

### Журнал решений

Настольные заголовки: Дата; Объект; Действие; Решение; Специалист; Организация. Причина, комментарий, запуск анализа и связь с предыдущим событием открываются в быстром просмотре события. На мобильном используются карточки.

## Состояния

- Загрузка данных
- Не удалось загрузить данные. Повторить
- По выбранным условиям данных нет
- Сравнение станет доступно после следующего запуска анализа
- Динамика появится после нескольких запусков анализа
- Сравнение недоступно: недостаточно сопоставимых организаций
- Этот сигнал обнаружен впервые. Предыдущая история отсутствует
- Повторяющиеся модели пока не сформированы
- Связанные сигналы недоступны в текущем запуске анализа
- Недостаточно экспертных решений для устойчивого вывода

Состояния показываются внутри соответствующего раздела и не занимают высоту полноценного аналитического блока.

## Устранённые дубли и противоречия

- Удалены отдельный пункт «Сигналы риска» и название «Очередь на проверку» в H1.
- Удалён блок «Следующее действие» на главной.
- «Причины сигнала» и «Сравнение» объединены в «Обоснование».
- Факторы важности, устойчивости и краткая динамика модели сведены в один раздел.
- Полная экспертная история скрыта на второй уровень; на первом видны последние действия.
- Общая история организации удалена; история доступна через конкретные сигналы и модели.
- Декоративные стрелки сортировки таблицы моделей удалены.

## Тон интерфейса

Система сообщает об отклонении, риске и необходимости экспертной оценки. Она не формулирует окончательный вывод за специалиста. Предпочтительны тексты: «выявлен сигнал риска», «обнаружено нетипичное отклонение», «рекомендуется дополнительная проверка», «результат требует оценки специалиста».
