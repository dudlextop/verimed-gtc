# Инвентаризация экранов Verimed

Дата актуализации: 13 июля 2026 года.

Основной пользователь — специалист по контролю медицинских услуг. Инвентаризация отражает интерфейс после этапа структурного упрощения.

|---|---|---|---|---|---|
| `/` | Понять, что требует внимания и что изменилось | Перейти к проверке | командный блок, изменения, приоритетные организации и модели | проверка, организации, модели | Оставить в сокращённом виде |
| `/overview` | Представить управленческий срез анализа | Перейти к приоритетной проверке | объём анализа, три приоритетных объекта, системные выводы, качество, модели, изменения | главная, проверка, организации, сигналы, модели | Отдельный полноэкранный экран без боковой навигации |
| `/organizations` | Выбрать организацию по приоритету | Открыть организацию | приоритет, риск, финансовая значимость, сигналы, динамика | организация, проверка | Упростить на следующем этапе |

## Основной маршрут

1. `/` — специалист видит объекты внимания.
2. `/signals` — сохраняет фильтры и выбирает сигнал.
3. Быстрый просмотр — оценивает приоритет без ухода из списка.
4. `/signals/{id}` — изучает четыре смысловых этапа.
5. Экспертное решение — статус обновляется без перезагрузки.
6. Действие «Перейти к следующему сигналу» — открывает следующий объект текущей страницы.
7. Возврат в `/signals` — фильтры, сортировка, страница и прокрутка восстанавливаются.

## Представление на разных ширинах

| Ширина | Поведение |
|---:|---|
| 1440 и 1280 px | Постоянная боковая панель, компактные таблицы, карточка с боковой навигацией по разделам. |
| 1024 px | Постоянная навигация; таблицы остаются настольными, но содержат только обязательные колонки. |
| 768 px | Навигация открывается поверх страницы; списки сигналов и моделей переходят в карточки. |
| 375 px | Одноколоночные карточки, один основной закреплённый призыв, граф ограничен по высоте, документ не должен быть шире viewport. |

Для `/overview` боковая панель не используется на любой ширине. На 1440 и 1280 px приоритетные объекты располагаются в одной строке; на меньших ширинах обзор получает обычную вертикальную прокрутку и последовательную компоновку.

## Изменения относительно аудита

- «Очередь на проверку» и «Сигналы риска» объединены в «Проверку».
- Результаты экспертной оценки и журнал объединены только на уровне группы навигации.
- Неактивные загрузка данных и настройки скрыты из навигации.
- Полная аналитика главной перенесена в закрытый блок второго уровня.
- Карточка сигнала сокращена с пяти до четырёх этапов.
- Карточка модели сокращена с семи до пяти разделов.
- История организации удалена как дублирующий общий блок.
