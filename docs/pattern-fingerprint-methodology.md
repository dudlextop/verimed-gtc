# Методика стабильного fingerprint модели

## Цель

Fingerprint связывает логически одну и ту же повторяющуюся модель между последовательными запусками анализа, даже если идентификаторы заново сформированных сигналов изменились.

## Состав

В каноническую структуру входят:

- технический тип модели;
- устойчивые идентификаторы ключевых организаций;
- устойчивые идентификаторы ключевых врачей;
- устойчивые идентификаторы ключевых услуг;
- код типа исходного отклонения, если он определяет модель;
- структура последовательности услуг;
- характеристика временного положения, например последние пять дней месяца;
- версия алгоритма fingerprint.

Идентификаторы отдельных медицинских записей и сигналов не включаются.

## Алгоритм

1. Признаки сортируются детерминированно.
2. Формируется JSON с сортировкой ключей и фиксированными разделителями.
3. UTF-8 представление хешируется SHA-256.
4. 64-символьная строка хранится в `recurring_patterns.fingerprint` с уникальным индексом.

Повторное совпадение увеличивает `recurrence_runs`, обновляет текущие связи и создаёт новый снимок. Экспертная история хранится независимо в `ExpertDecisionEvent` и находится по fingerprint, даже если текущая версия модели отсутствует.

## Версионирование

Текущая версия канонической структуры — `1`. Изменение состава признаков требует новой версии и миграционной стратегии сопоставления, чтобы не смешивать fingerprint с разной семантикой.
