# Локальный профиль специалиста

## Назначение и ограничение

Локальный профиль подготавливает инфраструктуру будущего ограниченного редактирования отображения. Он не является учётной записью, авторизацией или кадровым профилем. Данные сохраняются только в текущем браузере и не передаются backend.

При отсутствии или повреждении записи интерфейс использует воспроизводимый synthetic profile. Profile trigger читает локальные имя, должность и инициалы; без локальной записи его вид и содержание остаются прежними.

## Схема

Ключ localStorage: `verimed:local-profile`.

```json
{
  "version": 1,
  "data": {
    "displayName": "Айдана Сарсенова",
    "jobTitle": "Ведущий эксперт",
    "department": "Контроль медицинских услуг",
    "contactEmail": "a.sarsenova@example.local",
    "initials": "АС",
    "neutralAvatarPreset": "neutral"
  },
  "updatedAt": "2026-07-15T12:00:00.000Z"
}
```

Допустимые presets: `neutral`, `blue`, `cyan`, `teal`. Персональная фотография не хранится.

## Валидация и устойчивость

- имя: 2–80 символов;
- должность: до 80 символов;
- подразделение: до 100 символов;
- почта: до 160 символов, корректный email или пустое значение;
- инициалы: 1–3 русские, казахские или латинские буквы;
- пробелы нормализуются;
- частичное обновление сохраняет остальные валидные поля;
- safe parse, версия схемы, reset и fallback обрабатывают повреждённый и недоступный localStorage;
- событие `storage` синхронизирует изменения между вкладками.

Не сохраняются пароль, токены, права, административная роль, кадровые сведения, персональная фотография и чувствительные данные.

## Интеграция в профиль V2

Маршрут `/profile` предоставляет рабочую frontend-only форму для разрешённых полей схемы. Сохранение:

- выполняет ту же нормализацию и валидацию, что `validateLocalProfile`;
- показывает inline-ошибки и переводит фокус на первое некорректное поле;
- обновляет summary профиля и profile trigger без перезагрузки страницы;
- сохраняется после reload текущего браузера.

Reset удаляет ключ `verimed:local-profile` и возвращает исходный synthetic fallback. Это вторичное действие и не связано с удалением учётной записи.

Стандартный `storage` event синхронизирует другие вкладки. Для текущей вкладки store отправляет событие `verimed:local-profile-change`, потому что браузер не отправляет `storage` инициатору изменения.

Если localStorage недоступен, интерфейс продолжает показывать fallback-профиль, сообщает об ограничении и блокирует сохранение. Backend identity, экспертные решения и права доступа не изменяются.
